The Big Picture

How everything connects — from URL to database and back

A City Under the Streets

Every time you click a link or submit a form, invisible infrastructure springs into action — pipes you never see, working in sequence. Think of it like city infrastructure: the moment you turn on a tap, a whole system of reservoirs, pumps, and pipes delivers water — without you needing to know any of it exists.

The routing is the first pipe. The App Router connects URLs to code automatically. And cache is the stack of pre-built pages Next.js keeps ready so it doesn't start from scratch on every visit.

Here's the complete journey your data takes through a Next.js App Router app.

🌐
Browser

Where your user lives — the page they see and click

⚙️
Next.js Server

The traffic controller that routes every request to the right place

🧩
Server Components

Code that runs on the server, fetches data, and builds the page

🗄️
Database

Where all your data lives permanently — invoices, revenue, users

💡
Key Insight

In Next.js App Router, the line between frontend and backend is blurred. Server Components ARE the API — no separate API routes needed for most cases.

All the Patterns in One Place

Six major patterns run through every module. Here's the complete map of how the app is built.

🗂️

File-Based Routing

The URL is the folder. app/dashboard/page.tsx is /dashboard — no config needed.

⚡

Server Components

React components that run only on the server. Fetch data, query the database, return HTML. Default in App Router.

🎮

Client Components

Add 'use client' at the top of a file — this component runs in the browser and handles clicks, forms, and state.

🚀

Server Actions

Functions marked with 'use server' that run on the server and handle form submissions and database writes.

🗃️

SQL Template Literals

Raw SQL queries written as tagged template literals — the simplest way to talk to a Postgres database.

🔄

revalidatePath

A Next.js function that clears the cache for a route after a database write — so the next visit shows fresh data.

The debugging superpower

When something breaks, you'll know exactly where to look. Is data wrong? Trace back through the SQL query. Is the UI frozen? Check if a Client Component is missing 'use client'. Is old data showing? revalidatePath was probably forgotten.
