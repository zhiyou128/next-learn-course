Every Next.js project is a theater production. When you visit /dashboard, a cast of files springs to life — each one playing a specific role.

The Plot Twist

AI coding tools often put code in the wrong "room." Knowing which file does what means you can say "no, that's a UI piece — put it in the costume shop, not on stage."

🎭

The lead actor. Shows up on stage for one specific URL. Nothing else does its job.

👗

ui/ — The Costume Shop

The ui/ folder is where React components that are purely visual live. A button. A card. A navigation menu. Things any page might need.

CODE 
import {
  UserGroupIcon,
  HomeIcon,
  DocumentDuplicateIcon,
} from '@heroicons/react/24/outline';

const links = [
  { name: 'Home', href: '/dashboard', icon: HomeIcon },
  {
    name: 'Invoices',
    href: '/dashboard/invoices',
    icon: DocumentDuplicateIcon,
  },
  { name: 'Customers', href: '/dashboard/customers', icon: UserGroupIcon },
];

export default function NavLinks() {
  return (
    <>
      {links.map((link) => {
        const LinkIcon = link.icon;
        return (
          <a
            key={link.name}
            href={link.href}
            className="flex h-[48px] grow items-center justify-center gap-2 rounded-md bg-gray-50 p-3 text-sm font-medium hover:bg-sky-100 hover:text-blue-600 md:flex-none md:justify-start md:p-2 md:px-3"
          >
            <LinkIcon className="w-6" />
            <p className="hidden md:block">{link.name}</p>
          </a>
        );
      })}
    </>
  );
}

The Reuse Principle

Because NavLinks lives in ui/, the dashboard layout can import it once and every page in the dashboard gets the same nav — without duplicating any code.

lib/ — Backstage Crew

While ui/ holds things you can see, lib/ holds things you use. Utility functions, data transformers, API helpers. The backstage crew that the cast calls on when they need something.

1
A page needs to format a dollar amount

Instead of writing the formatting logic in the page, it imports a utility function from lib/utils.ts

2
The page focuses on what to display

It hands the raw number to the utility function and gets back a nicely formatted string like $1,234.56

3
Any other page can use the same function

Consistency across the whole app — every dollar amount formats the same way, maintained in one place.

Cast Glossary

Every term from this module, defined plainly.

page.tsx
The lead actor of each URL. When someone visits /dashboard, app/dashboard/page.tsx runs and returns the HTML.
layout.tsx
The stage manager. Wraps a group of pages with shared navigation, sidebars, or footers so you don't have to repeat them.
ui/
The costume shop. Contains visual components — buttons, cards, nav menus — that any page can import and reuse.
lib/
Backstage crew. Holds shared logic, formatting helpers, and data-transforming functions that any page can call on.
React component
A reusable piece of UI — like a button, card, or nav menu. You write it once and drop it into any page that needs it.
utility function
A helper function that does a specific job — like formatting a date or converting a number to currency. Pages import it when they need it.
@/ alias
A shortcut for the app directory. Instead of writing ../../ui/button, you write @/ui/button. Less typing, same result.
