服务器 vs 客户端 — 代码在哪里运行？

有些代码在后厨（服务器）运行，有些在你桌边（浏览器）运行

每个应用都有两个用餐区

🏠
前台 — 浏览器（客户端）

用户坐的地方。他们看菜单、点击按钮、在表单中输入。快速且交互性强——但他们无法访问数据库或密钥。

👨‍🍳
后台 — 服务器

食物制作的地方。可以访问一切——数据库、API、密钥。但离食客很远，所以对点击的响应较慢。

💡
关键问题

当你点击一个按钮时，决策是在你的桌边（浏览器）还是厨房（服务器）做出的？把代码放错地方会悄悄破坏功能。

服务器组件是默认选项

默认情况下，Next.js 中的每个服务器组件都在服务器上运行。不会为其发送任何客户端 JavaScript。

📊

显示数据

从数据库获取数据、读取文件、调用 API——在服务器上做这些都很安全，密钥不会暴露。

🎨

纯 UI

静态布局、字体排版、设计系统——任何不需要用户交互的渲染。

🔒

安全敏感

任何使用 API 密钥、密码或不应该离开服务器的业务逻辑。

🗄️
须知

你可以通过没有特殊标记来识别服务器组件——因为它是默认的。不需要指令。

当你需要交互性时：'use client'

有些组件需要响应点击、跟踪输入或与浏览器 API 通信。这时你需要添加 'use client' 指令。

CODE 
'use client';
// This file runs in the BROWSER, not on the server

import { lusitana } from '@/app/ui/fonts';
import { AtSymbolIcon } from '@heroicons/react/24/outline';

export default function LoginForm() {
  // useState and event handlers live HERE
  return (
    <form className="space-y-3">
      {/* form fields with onClick, onChange */}
    </form>
  );
}
PLAIN ENGLISH

'use client' 告诉 Next.js：「把这代码发送到浏览器。」

注释是给你的：让你一眼就知道这个文件在哪里运行。

导入字体：和服务器组件版本一样——这里没有区别。

函数是一样的——但现在它可以使用 useState、onClick 处理程序等。

JSX 标记：表单本身看起来一样。只有能力不同。

💡
关键洞察

默认情况下，Next.js App Router 中的所有组件都是服务器组件。只有当你需要交互性——点击处理程序、状态（useState）或浏览器 API 时，才添加 'use client'。

什么真正发送到浏览器？

不是所有代码都会到达用户。以下是服务器组件保留给自己的内容。

// This stays on the server — NEVER reaches browser
const dbPassword = "super-secret-key-123";
await fetchRevenue();

// Database queries
const customers = await db.query("SELECT * FROM users");
// HTML and CSS arrive — the rendered result
<div class="revenue-chart">
  <h2>Monthly Revenue</h2>
  <svg>...chart...</svg>
</div>

/* CSS styles */
// ONLY LoginForm's JS bundle ships
'use client';
export default function LoginForm() {
  const [state, setState] = useState();
  return <form>...</form>;
}

服务器组件将密钥、数据库和重型逻辑保留在服务器上。浏览器只收到 HTML、CSS 和交互式组件的 JavaScript。

服务器 vs 客户端：谁来处理点击？

观察当用户点击按钮时会发生什么——每个组件都认为应该由另一个来处理。

S
Server Component

我准备好了数据。按钮应该只显示我发送的内容。

C
Client Component

没那么快。点击事件发生在浏览器里——那是我的领域。

S
Server Component

没错——但我可以在页面加载前预取数据。然后你只管显示就行。

C
Client Component

成交。你做重活（数据库）。我处理有趣的部分（点击、悬停、表单）。

?

这种分工很自然：服务器组件获取并准备数据。客户端组件处理事件处理程序和状态。

何时画这条线

1

它需要数据库吗？

→
2

它使用密钥或 API 密钥吗？

→
3

它响应点击或输入吗？

→
4

它使用仅浏览器的 API 吗？

步骤 1-2 → 服务器组件（默认）。步骤 3-4 → 'use client'。

fetchRevenue() 服务器组件 — 数据库访问，不向浏览器发送 JavaScript
onClick={handleSubmit} 客户端组件 — 事件处理程序需要 'use client'
useState, useEffect 客户端组件 — 这些是只在浏览器中工作的钩子
window.innerWidth 客户端组件 — 服务器上不存在的浏览器 API
